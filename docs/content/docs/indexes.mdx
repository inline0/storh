# Indexes

`DocStore` indexes are stored beside the collection under `.storh/indexes`.
They are ordinary files and can be rebuilt from records.

```php
$docs->indexes()
    ->field('slug')->unique()
    ->field('kind')
    ->field('publishedAt')->range()
    ->sync();
```

Index types:

- equality indexes for fields like `kind` or `status`
- unique indexes for fields like `slug`
- range indexes for numeric, string, boolean, and null scalar values

Equality and unique indexes are split by field value, so point lookups read only
the matching value file. Writes update index entries after the record file is atomically replaced.
Deletes remove matching index entries. `reindex()` rebuilds every configured
index from the record files.

When a query combines two `eq` predicates on non-unique equality-indexed fields,
storh can use an automatic compound bucket for that field pair. This keeps
selective two-field filters fast without requiring another index declaration.

```php
$docs->reindex();

$plan = $docs
    ->query()
    ->where('slug')->eq('home')
    ->explain();
```

`explain()` returns `index_scan` when the planner can use a configured index and
`full_scan` otherwise.
